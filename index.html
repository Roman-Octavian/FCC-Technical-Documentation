What is HTML?

HTML or HyperText Markup Language is a markup language that tells web browsers how to structure the web pages you visit. HTML consists of a series of elements, which you use to enclose, wrap, or mark up different parts of content to make it appear or act in a certain way.

"Hypertext" refers to links that connect web pages to one another, either within a single website or between websites. HTML uses "markup" to annotate text, images, and other content for display in a Web browser. An HTML element is set off from other text in a document by "tags", which consist of the element name surrounded by "<" and ">".

What do you need?

To start working with HTML, there are a few requirements:

  • Basic computer literacy
  • Basic software installed on your machine
  • Basic knowledge of working with files

Notwithstanding that there are many ways to learn HTML, certain online resources such as freeCodeCamp or CodePen will allow you to learn and write HTML straight from your browser.

Basic structure of HTML

HTML elements are as seen in the image below mainly comprised of opening tags, optional attributes, the content itself, and closing tags.

The basic structure of an HTML file normally includes the <!DOCTYPE> at the very top, the <html> element right below encasing the rest of the code, and the <head> and <body> respectively. This basic structure would look something like this:

<!DOCTYPE html>
<html>
    <head>

    </head>
    <body>

    </body>
</html>

The <head> should contain a <title>, whereas the body has a structure of its own that usually contains:

  • a <header>
  • a <main> tag where the main content goes
  • a <footer> at the bottom of the page
HTML Metadata

The <head> is not displayed in the browser when the page loads as it mostly contains metadata. Metadata is data about the HTML itself. The most common elements included in the <head> are:

  • The <title> element, containing the overall title of the HTML, not to be confused with the <h1> heading.
  • Many <meta> tags which add a description, an author, images, theme colors, etc.
  • A favicon included with a <link> element.
  • And many others.

HTML fundamentals

To insert basic text, we use the <p> tag like so:

<p>I am a paragraph!</p>

To add headers, we use the <h> tags. They go from one to six. It is important to use them in a proper hierarchical order as they are supposed to give structure to the page.

<h1>Your title goes here</h1>
    (content)
    <h2>Section</h2>
    (content)
        <h3>Subsection</h3>

You can also create either ordered or unordered lists with the <ol> and <ul> tags like so:

<ul>
    <li>(list item 1)</li>
    <li>(list item 2)</li>
    <li>(list item 3)</li>
    <li>(list item 4)</li>
</ul>

Output:

  • (list item 1)
  • (list item 2)
  • (list item 3)
  • (list item 4)

Finally, you can style text with the <em> and <strong> tags. The first represents emphasis and the second strong importance.

<p>You <em>must</em> eat your veggies!</p>
<p>Do <strong>NOT</strong> try this at home.</p>

Equals:

You must eat your veggies!

Do NOT try this at home.

Creating hyperlinks

Hyperlinks allow us to link documents to other documents or resources, link to specific parts of documents, or make apps available at a web address.

To create a link, we use the anchor element like this:

<a href="URL, file path, or ID attribute goes here">Link text</a>

You can put anchors within other elements such as images or paragraphs. For example:

<p>Click on the following <a href="https://www.google.com/" target="_blank">link</a> to open Google.</p>

Equals:

Click on the following link to open Google.

Note: target="_blank" indicates that the link should open in a new tab.

Images in HTML

Adding images to HTML is quite easy. We use the <img> tag like this:

<img src="image source goes here">

Adding an alt attribute to your image is mandatory. This alternative text will appear if for some reason your image fails to load, hence why its value should be a textual description of the image it accompanies.

You can specify the width and height of an image as well, and you can also add an optional title that will appear when you hover over the image. Titles however aren't ideal because only certain devices display them properly.

Here's an example:

<img src="images/html.png" alt="The HTML5 Logo" width="300" height="300" title="The HTML5 Logo">

Videos and Audios

Embedding Videos and Audios to your HTML works in a similar fashion to how images do. We use the <video> and <audio> tags.

Here's how they would look like:


<video controls width="400" height="400" preload="auto">
<source src="videos/rabbit320.mp4"> type="video/mp4";
<p>Video failed to load. Open <a href="videos/rabbit320.mp4">this</a> URL instead to watch.</p>
</video>

<audio controls preload="auto">
<source src="audios/viper.ogg" type="audio/ogg">

HTML comments

Comments are a powerful tool in HTML. They allow you to make annotations directly into the source code without affecting it's functionality unless you want it to. Thus, comments can be used to tell others, or even yourself if you're forgetful, what certain sections of your HTML do. Wrapping code in comments also disable it which can be useful if you want to temporarily deactive a part of your code without deleting it.

Here's how a comment looks like:

<!-- Below lies a very important paragraph! -->
<p>Very important paragraph.</p>

Equals:

Very important paragraph

As expected, you cannot see the comment.

Applying CSS and JS

To style your HTML, you'll most likely want to use Cascading Style Sheets abbreviated CSS. There are three ways to apply CSS to your HTML:

  • By coding it directly into the element with the <style> attribute
  • By adding a <style> section to your HTML
  • By using the <link> attribute to reference an external file.

While the first two methods are OK for simple webpages, CSS quickly occupies lots of space so the third option is the most common amongst developers. To link to an external CSS file, all you have to do is link to it in the <head> of your HTML like so:

<link rel="stylesheet" href="your-css-file-name.css">

Applying JavaScript is easy too. All you have to do is link to it with the <script> element like this:

<script src="URL or local path goes here"></script>
Debugging HTML

Whilst writing HTML you'll most likely make mistakes every once in a while. Since HTML is a permisive language, your webpage will still load in spite of said mistakes. However, the result may not be acceptable. To mend this, you ought to debug your HTML, to find out what went wrong and fix it. There are many ways to debug HTML, including:

  • Reviewing your HTML manually, focusing on the sections that appear to be broken looking for missing tags, misspellings and other errors.
  • Making use of debug tools built into a code editor. If you use a code editor such as Visual Studio Code, there are debug tools available to you, and the program itself will usually make it pretty obvious when there is a mistake.
  • Using a validator such as the W3 Validator. This tool made by the organization responsible for HTML itself will take your HTML and look for errors.

Most environments in which you'll be working with HTML have "undo" buttons so as long as you back-up your code, trial and error and messing around should be enough to fix most bugs.
